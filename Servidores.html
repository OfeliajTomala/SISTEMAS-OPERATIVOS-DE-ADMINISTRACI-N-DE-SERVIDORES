SERVIDOR DNS

¿Qué es y para qué sirve?

Un servidor DNS (Domain Name System) actúa como la guía telefónica de Internet. Traduce nombres de dominio fáciles de recordar (como www.google.com) en direcciones IP numéricas (como 142.250.190.78), lo que permite que los navegadores localicen y accedan a los sitios web correctos. Funciona mediante varios tipos de servidores: recursivos, raíz, de dominio de nivel superior (TLD) y autorizados, que colaboran en el proceso de resolución


Como funciona:

  1. Tu dispositivo envía la solicitud al servidor DNS recursivo.
  2. El servidor verifica su caché o consulta a los servidores raíz y de nivel superior (TLD).
  3. Llega al servidor autorizado que conoce la IP del dominio.
  4. Devuelve la IP al dispositivo para que el navegador se conecte al sitio.

Ejemplo

Cuando escribes “www.wikipedia.org” en tu navegador, el servidor DNS traduce ese dominio a su dirección IP, por ejemplo 208.80.154.224, permitiendo que tu navegador se conecte al servidor web de Wikipedia.


SERVIDOR WEB

¿Qué es y qué función cumple?

Un servidor web es tanto un software como el hardware que almacena, procesa y entrega contenido (páginas web, imágenes, scripts) a los navegadores mediante el protocolo HTTP o HTTPS


Como funciona:

  1. El navegador hace una petición HTTP/HTTPS al servidor.
  2. El servidor procesa la solicitud (puede ejecutar scripts o consultar bases de datos).
  3. Envía los archivos necesarios (HTML, CSS, JS, imágenes) al navegador.
  4. El navegador interpreta los archivos y muestra la página.

Ejemplo

Sitio web: Amazon.
El servidor entrega las páginas de productos, imágenes y detalles de compra a tu navegador.


SERVIDOR FTP

¿Para qué se usa?

Un servidor FTP (File Transfer Protocol) permite la transferencia de archivos entre un cliente y un servidor usando dos canales TCP/IP: uno para comandos y otro para datos.No cifra los datos, por lo que su uso en entornos sensibles está en desuso a favor de variantes más seguras como SFTP.


Como funciona:

  1. El cliente FTP se conecta al servidor con usuario y contraseña.
  2. Se establece un canal de comandos y otro de datos.
  3. Puedes subir archivos al servidor o descargar archivos desde él.

Ejemplo

Un diseñador gráfico sube archivos de una campaña publicitaria a Adobe Creative Cloud mediante FTP para compartirlos con su equipo.


SERVIDOR DHCP

¿Qué hace con las direcciones IP?

El servidor DHCP (Dynamic Host Configuration Protocol) asigna automáticamente direcciones IP y otros parámetros de red a dispositivos conectados. Esto evita configuraciones manuales y facilita la conexión automática de equipos a redes tanto domésticas como empresariales


Como funciona:

  1. Un dispositivo solicita conectarse a la red.
  2. El servidor DHCP asigna una IP disponible y parámetros de red.
  3. La IP tiene un tiempo de arrendamiento que se renueva automáticamente.

Ejemplo

En casa, tu router asigna automáticamente IPs a tu computadora, tablet y teléfono al conectarse al Wi-Fi.


SERVIDOR DE CORREOS:

Diferencia entre enviar (SMTP) y recibir (IMAP/POP3)

  • SMTP (Simple Mail Transfer Protocol): Envía correos desde tu cliente a otros servidores.
  • IMAP (Internet Message Access Protocol): Mantiene los correos en el servidor y sincroniza los cambios en varios dispositivos.
  • POP3 (Post Office Protocol versión 3): Descarga los correos al dispositivo y, por defecto, los elimina del servidor

Como funciona:

  1. SMTP envía el mensaje desde tu cliente al servidor de destino.
  2. IMAP permite leerlo y sincronizarlo en varios dispositivos.
  3. POP3 descarga el mensaje y lo almacena localmente en un solo dispositivo.

Ejemplo

Envías un correo usando Outlook: SMTP envía el correo, IMAP permite que tu colega lo lea en su computadora y móvil, POP3 lo descargará solo a un dispositivo si se usa esa opción.
